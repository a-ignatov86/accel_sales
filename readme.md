## Область и краткое описание задачи
Создать устойчивую, воспроизводимую и документированную ML-систему, которая по входным историческим данным (продажи/заявки/транзакции + календарно-метео/маркетинг/акции) выдаёт прогнозы спроса на заданный горизонт (напр., 7/14/30 дней) с объяснимостью предсказаний и мониторингом качества в эксплуатации.


## Данные

продажи: timestamp, sku_id (или service_id), location_id, quantity, price, order_id, channel.
Каталожные данные: sku_id, category, sub_category, pack_size, lead_time.
Календарь: праздники (локальные/национальные), рабочие/выходные дни.
Маркетинг: кампании (start_date, end_date, type, discount_pct).
Внешние факторы: погода (температура, осадки), экономические индикаторы, события.


Очистка и предобработка

Обрабатывать пропуски в quantity и price (логика заполнения описать и согласовать).

Удаление дубликатов.

Выявление и пометка аномалий — с указанием логики.

Подготовить feature-engineering pipeline (бинаризация и категоризация - если необходимо, скользящие средние, лаги, оконные статистики, календарные признаки, промо-флаги, погодные признаки).

## API для получения прогнозов

REST API (Flask/FastAPI) с минимум следующими эндпоинтами:

POST /predict — принимает JSON с sku_id, location_id (опц.), start_date, horizon, дополнительные фичи; возвращает прогноз по дням + интервалы.

POST /predict/batch — загрузка CSV/Parquet для прогнозов.

GET /health — статус сервиса.

# План для презентации
## 2 Описать проблему 
Коммерческим компаниям важно уметь предсказывать объемы спроса на продукцию, чтобы избежать затаваривания, использования избыточных складских помещений, либо недопроизводства товаров при наличии повышенного спроса.   Без прогнозов склад и транспорт часто перегружены в пиковые периоды и простаивают в низкие. Ошибки в планировании могут привести к излишним затратам или снижению потенциальной выручки.

## 3 Описать предлагаемое решение (конечный продукт) для обозначенной проблемы. Указать уникальность (инновационность) вашего решения и область его применения, с обязательным указанием технологической составляющей стартап-проекта

Модель машинного обучения прогнозирует спрос с учётом сезонности, трендов и событий. Это позволит компании заранее знать, какие объёмы и позиции потребуются, и поможет заранее планировать пики загрузки складских помещений, оптимизировать графики отгрузок товара торговым точкам или покупателям, распределять ресурсы по складам и по времени.

Система  позволяет моделировать сценарии изменения спроса при изменении цены, введения скидкок, маркетинговых промокомпаний, а также покажет какие категории товара наиболее чувствительны к маркетинговым действиям.

инновационность нашего решения заключается в том, что ML-модель позволяет автоматизировать декомпозицию временного ряда, а также интегрирует внешние факторы, такие как промоакции, календари, макроэкономическую статистику при необходимости. 

При создании проекта используется язык программирования Python, библиотеки Pandas для работы с датафреймом, mysql.connector для работы сбазами данных, Matplotlib для визуализации результатов, scipy для работы со статпакетами, prophet для анализа временных рядов.

Результаты

Снижение излишков на складе
Более равномерная загрузка персонала и транспорта
Повышение точности управленческих решений
Сокращение дефицита и отказов клиентам 
Освобождение оборотных средств.
