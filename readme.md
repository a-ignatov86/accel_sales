## Область и краткое описание задачи
Создать устойчивую, воспроизводимую и документированную ML-систему, которая по входным историческим данным (продажи/заявки/транзакции + календарно-метео/маркетинг/акции) выдаёт прогнозы спроса на заданный горизонт (напр., 7/14/30 дней) с объяснимостью предсказаний и мониторингом качества в эксплуатации.


## Данные

продажи: timestamp, sku_id (или service_id), location_id, quantity, price, order_id, channel.
Каталожные данные: sku_id, category, sub_category, pack_size, lead_time.
Календарь: праздники (локальные/национальные), рабочие/выходные дни.
Маркетинг: кампании (start_date, end_date, type, discount_pct).
Внешние факторы: погода (температура, осадки), экономические индикаторы, события.


Очистка и предобработка

Обрабатывать пропуски в quantity и price (логика заполнения описать и согласовать).

Удаление дубликатов.

Выявление и пометка аномалий — с указанием логики.

Подготовить feature-engineering pipeline (бинаризация и категоризация - если необходимо, скользящие средние, лаги, оконные статистики, календарные признаки, промо-флаги, погодные признаки).


# Метрики качества
R2
MAE/RMSE
MAPE (Mean Absolute Percentage Error)



## API для получения прогнозов

REST API (Flask/FastAPI) с минимум следующими эндпоинтами:

POST /predict — принимает JSON с sku_id, location_id (опц.), start_date, horizon, дополнительные фичи; возвращает прогноз по дням + интервалы.

POST /predict/batch — загрузка CSV/Parquet для прогнозов.

GET /health — статус сервиса.


# План работ и вехи 

Веха 1 (1–2 недели): Сбор и анализ данных (EDA), согласование формата данных, подготовительный отчет.

deliverables: EDA notebook, список полей, предложенная схема агрегирования, очистки и признаки.

Веха 2 (1–2 недели): Разработка продвинутой модели(ей), кросс-валидация, подбор гиперпараметров.

deliverables: обученные модели, backtest отчёты.

Веха 3 (1–2 недели): API и деплой в тестовую среду, интеграционные тесты.

deliverables: docker image, API endpoints, примеры запросов.
